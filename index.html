自制markdown语法说明

发表于 2023-04-01

段落：

  1. “—-” ： 分割线如上
  2. “两个空格加换行” ： 换行
  3. “中间隔一个空行” ：创建段落

字体

  1. 粗体 ： 四个星号中间包围
  2. 斜体 ： 两个星号中间
  3. 粗体加斜体 ： 三个星号中间包围着

引用

  1. 引用块 ： “>”在大于号后面添加内容即可
  2. 多段落引用： “>”在段落与段落之间的空行上面也加入“>”
  3. 引用格式：引用块中同样遵循Markdown格式
  4. 文内链接：***作者的网站***,【】（）中括号里面是文本内容，小括号是链接地址 ，给出链接地址之后可以加“”,在双引号里编辑提示。可以被星号强调以及斜体

文内格式

  1. 列表 ： 数字列表直接输入数字“1.”，并在小数点后加空格

  2. 无序列表：用“-”代替“1.”

  3. 在列表中保留元素：加入一个tab缩进

    • 嵌套列表：加入tab缩进之后继续使用“1.”或者“-”

  4. 代码块：直接缩进四个空格，如果在列表中，缩进8个空格,与正文中间留空行 

     #include<stdio.h>  
 int  main(void)  {  
     printf("hello,world!");
     return 0;
 }

  5. 单词或短语转为代码：“code time”,用反引号标注起来

  6. 图片：要添加图像，请添加感叹号（!），然后在中括号中添加替代文本，并在小括号中添加图像资源的路径

  7. 转义字符： 显示Markdown语法所占用的原有的字符，“/”后面添加即可，”\ *” = *

第一篇博客

发表于 2023-04-01

为什么有这篇博客

emmm，起因是在b乎上面刷到了别人做的博客网站，感觉很酷炫，很有意思，自己也是将来要成为程序猿的人，为什么不自己做一个呢？网上的教程也有很多。我为了简单一点，能够满足自己日常使用就可以了，于是我选择用Hexo加上git以及npm来搭建这个框架，截止到目前为止（3.31），整体还是十分简陋，但是有一个以自己名字命名并且可以畅所欲言的地方的感觉已经让我十分迷恋了！这种感觉不同于任何社交平台，因为理论上来说，你的一切数据以及账户以及发表在该平台下的言论，以及与他人的互动也许可能有一天就会随着该平台的倒下而消失。而且我本身是个不爱表达的人，因为潜意识里总是担心被不怀好意者窥探，个人博客倒是让我感觉没有了那么多的束缚。希望能越做越好，起码这是我的第一个尝试，以后可能会有功能更完善，界面更漂亮的博客，但是这个是我的第一个精神家园。~~~~撒花~~~~~~

一些看法

博客会不会过时？

应该不会吧，毕竟感觉每个博客都凝聚了每个人的心血，不会轻易放弃的。再者，现在公共博客平台上的内容属实是太多太杂，导致一些真正优质的博客很难被找到，经常性的问题是，我被某个问题困住了，在搜索了诸多平台之后才能找到一个合适的解决方案，而下次犯同样的错误时，也没有记录。折腾来去也会很浪费时间。 即使现在朋友圈和微博qq空间等分享的平台日益增多，我觉得博客还是一个比较不错的分享方式，期待一下吧。自己在上面写的一些简单易懂的知识也可以随时查看。
